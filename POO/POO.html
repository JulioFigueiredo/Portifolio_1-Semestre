Classe

É um conceito muito importante em POO, pois a classe é uma abstração de um objeto, é como se fosse uma forma, e a partir dela é possível instanciar vários objetos.

A estrutura de uma classe contém:

  • Nome da classe
  • Atributos
  • Métodos

Encapsulamento

Outro conceito muito importante em POO, que tem o objetivo de ocultar alguns detalhes de uma classe.


Em Java, existem 3 modificadores de acesso:
  • public: o atributo ou método pode ser acessado de qualquer lugar, sendo de sua classe, classes do mesmo pacote ou de pacotes diferentes.
  • private: um atributo ou método privado só pode ser acessado dentro de sua classe.
  • protected: um atributo ou método protegido só pode ser acessado dentro de sua classe ou de classes do mesmo pacote.

Métodos:

Construtor

O objetivo deste método é inicializar os atributos de um objeto quando for instanciado

                
                    public class Carro {
                        // atributos
                        private String nome;
                        private String marca;
                        private int ano;
                        private double kilometragem;
                    
                        // construtor
                    
                        public Carro(String nome, String marca, int ano, double kilometragem) {
                            this.nome = nome;
                            this.marca = marca;
                            this.ano = ano;
                            this.kilometragem = kilometragem;
                        }

Getters and Setters

O objetivo desses métodos é fazer com que os usuários da classe modifiquem os atributos de maneira controlada.
Normalmente as IDEs tem alguma função para gerar ambos os métodos. 

                
                    public String getNome() {
                        return nome;
                    }
                
                    public void setNome(String nome) {
                        this.nome = nome;
                    }
                
                    public String getMarca() {
                        return marca;
                    }
                
                    public void setMarca(String marca) {
                        this.marca = marca;
                    }
                
                    public int getAno() {
                        return ano;
                    }
                
                    public void setAno(int ano) {
                        this.ano = ano;
                    }
                
                    public double getKilometragem() {
                        return kilometragem;
                    }
                
                    public void setKilometragem(double kilometragem) {
                        this.kilometragem = kilometragem;
                    }

ToString

Este método é utilizado para descrever o objeto por completo em seu estado atual, facilitando a visualização do usuário. 

                
                    @Override
                    public String toString() {
                        return "Carro{" +
                                "nome='" + nome + '\'' +
                                ", marca='" + marca + '\'' +
                                ", ano=" + ano +
                                ", kilometragem=" + kilometragem +
                                '}';
                    }

Herança

É um conceito muito importante em Orientação a Objetos, que permite que uma classe (filha/subclasse), herde atributos e métodos do (pai/superclasse), fazendo com que não seja necessário repetição de código.

Nesse caso, Shape é a classe pai, Triangle, Rectangle e Circle herdam de Shape e Square herda de Rectangle.
Segue o exemplo para implementação em Java: 

                
                    public class Shape{

                    }
                
                
                    public class Rectangle extends Shape{

                    }

Métodos Sobrescritos

Em herança, a subclasse herda métodos da superclasse, porém há como substituir o método, para isso, em Java, é utilizado a notação @Override

Classes e métodos abstratos

Classes abstratas não podem ser instanciadas. No exemplo acima, a classe Shape pode ser uma classe abstrata, pois é a generealização das formas geométricas, assim, não precisa que seja instanciada diretamente. Em Java, é utilizado a palavra "abstract"

Métodos abstratos são declarados em classes abstratas, porém tem que ser substituidos na subclasse, ele é útil para que ele não seja esquecido na hora de implementar no projeto. A sintaxe é "abstract".

Polimorfismo

Outro conceito importante em POO, o Polimorfismo permite que uma variável refira-se a diferente tipos de objetos
No exemplo, a variável do tipo Shape (superclasse) aponta para objetos de diferente tipos (subclasses Rectangle, Square, Ellipse e Circle) 

                
                    Shape shape = new Rectangle();
                    shape = new Square();
                    shape = new Ellipse();
                    shape = new Circle();
